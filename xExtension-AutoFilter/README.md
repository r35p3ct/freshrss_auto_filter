# FreshRSS Auto Filter Extension

Расширение для автоматического определения и фильтрации рекламных записей в FreshRSS с использованием AI (OpenRouter API).

## Возможности

- **Автоматическое определение рекламы** с помощью AI (OpenRouter API)
- **Гибкая система меток**:
  - `Реклама` — высокая уверенность (>80%)
  - `Подозрение` — средняя уверенность (>50%)
  - `Непроверено` — запись ожидает фоновой проверки
- **Фоновый режим проверки** — записи добавляются мгновенно, AI-проверка идёт отдельным скриптом
- **Настраиваемые пороги уверенности** для точной настройки фильтрации
- **Валидация конфигурации** — автоматическая проверка корректности настроек
- **Безопасное логирование** — метаданные вместо полного содержимого записей

## Установка

1. Скопируйте папку `xExtension-AutoFilter` в директорию расширений FreshRSS:
   ```
   ./extensions/xExtension-AutoFilter/
   ```

2. Включите расширение в настройках FreshRSS:
   - Перейдите в `Настройки` → `Расширения`
   - Найдите `Auto Filter` и нажмите `Включить`

3. Создайте необходимые метки в FreshRSS:
   - `Реклама`
   - `Подозрение`
   - `Непроверено`
   - `Проверено`

## Настройка

### API ключ

Получите API ключ на [openrouter.ai/keys](https://openrouter.ai/keys)

### Модель

Рекомендуемые модели:
- `openai/gpt-3.5-turbo` — быстро и недорого
- `openai/gpt-4` — более точное определение
- `anthropic/claude-3-haiku` — хорошая альтернатива

### Пороги уверенности

- **Порог для рекламы** (по умолчанию 0.8) — при уверенности выше этого значения запись считается рекламой
- **Порог для возможной рекламы** (по умолчанию 0.5) — при уверенности выше этого значения запись помечается как возможная реклама

### Фоновый режим (рекомендуется для API-клиентов)

Если вы используете CapyReader, Fluent Reader или другие клиенты через Fever/GReader API и сталкиваетесь с задержками при обновлении лент:

1. Включите **Фоновый режим проверки** в настройках расширения
2. Настройте размер пачки (сколько записей проверять за раз, по умолчанию 5)
3. Настройте задержку между запросами (по умолчанию 2000 мс) — помогает избежать rate limit
4. Добавьте в cron задачу для запуска фонового скрипта:

```bash
# Каждые 5 минут
*/5 * * * * php /path/to/freshrss/extensions/xExtension-AutoFilter/scripts/process_pending.php
```

Для Docker:
```bash
*/5 * * * * docker exec freshrss php /var/www/FreshRSS/extensions/xExtension-AutoFilter/scripts/process_pending.php
```

**Как это работает:**
- При обновлении лент новые записи мгновенно добавляются с меткой `Непроверено`
- Фоновый скрипт постепенно проверяет их через AI и заменяет метку на результат (`Реклама`, `Подозрение` или удаляет)
- При чтении через API-клиенты новости отдаются мгновенно, без задержек на AI-проверку

### Промт по умолчанию

Расширение использует оптимизированный промт с чётким форматом вывода JSON. Вы можете задать собственный промт, используя плейсхолдеры:
- `{title}` — заголовок записи
- `{author}` — автор
- `{url}` — ссылка
- `{content}` — содержимое

### Логирование

При включённом логировании записываются:
- Метаданные промпта (длина, количество авторов)
- Краткий preview ответа API (первые 100 символов)
- Результат анализа (метка, уверенность, причина)

> **Примечание:** Полное содержимое записей не логируется для защиты конфиденциальности.

## Использование

### Автоматическая проверка (синхронный режим)

Расширение автоматически проверяет новые записи при их получении через хук `entry_before_add`.

> **Внимание:** В синхронном режиме каждая новая запись блокирует добавление до завершения AI-запроса. Это может вызывать задержки при чтении через API-клиенты.

### Ручная проверка

1. Найдите кнопку 🛡️ в заголовке записи
2. Нажмите для проверки записи на рекламу
3. Запись будет помечена соответствующей меткой

### Фильтрация по меткам

Используйте встроенные средства FreshRSS для фильтрации:

1. Создайте ярлык с фильтром по метке:
   - `label:Реклама` — показать только рекламу
   - `-label:Реклама` — скрыть рекламу
   - `label:Подозрение` — показать возможные рекламы

2. Используйте ярлыки для быстрого доступа к отфильтрованным записям

## API

### Проверка одной записи

```
GET /api/p.php?c=autoFilter_openrouter&a=checkEntry&entry_id={ID}
```

Ответ:
```json
{
    "success": true,
    "entry_id": 123,
    "analysis": {
        "is_advertisement": true,
        "confidence": 0.95,
        "reason": "Коммерческое продвижение товара",
        "label": "Реклама"
    }
}
```

### Массовая проверка

```
POST /api/p.php?c=autoFilter_openrouter&a=checkBatch
{
    "entry_ids": [123, 456, 789]
}
```

## Требования
